JAVASCRIPT AND THE DOM

24 January 2024

JavaScript and its relationship to HTML and CSS

Imagine you're building a car. HTML is like the frame and body of the car â€” it gives the car its basic structure and shape. Without a solid frame, the car wouldn't have a proper foundation to build upon.

CSS, on the other hand, is like the paint job, the wheels, and everything that gives the car its 'look'. It makes the car look good, but it doesn't make it run.

Now, JavaScript is the engine. It's what makes the car actually move, and change speed when you press the gas pedal. Without JavaScript, the car would just sit there, looking good but not going anywhere. JavaScript brings the interactivity and functionality.

Control Flow and Loops in JavaScript

Control flow is the order in which a program runs. It's like following a set of instructions that can change depending on the situation. For example, you might tell your program to do one thing if a condition is true, and something else if it's not. It's like asking: "If it's raining, take an umbrella. If it's sunny, wear sunglasses."

Loops are helpful when you need to repeat a task multiple times. Say you have a list of items you want to display on a webpage, you could use a loop to show each item without having to write the same code over and over. Instead of telling the program to do something for each item manually, a loop allows it to repeat the task automatically for every item in the list.

The Document Object Model (DOM)

Now that we know how JavaScript interacts with HTML and CSS let's talk about the DOM! When building a webpage, HTML, CSS, JavaScript, and the DOM each have an important role in bringing everything together.

The DOM (Document Object Model) acts as a bridge between the webpage and JavaScript. It's an internal, live representation of your HTML structure that the browser uses. When the webpage loads, the DOM is created based on your HTML content. JavaScript then interacts with the DOM to manipulate the page.

Let's say you have a button on a page and when you click it the background colour of the page changes. In the background HTML, CSS, JavaScript, and the DOM are all working together to make this happen.

  • The button is defined in HTML and given the id 'myButton'.
  • The colour of this button and the size of the button is styled with CSS. Inside the CSS we'll also have the colour of the typical background and the colour of the background when it's pushed.
  • JavaScript is listening for the click using .addEventListener() and runs a function to change the background colour of the page when the button is clicked.
  • JavaScript is using the DOM to find the button and change the background colour. Using document.querySelector('#myButton'), JavaScript can target the button, then update the background colour when the button is clicked.

How to access data from arrays and how to access data from objects

Firstly, what's the difference between arrays and objects? Well they're both ways of storing data but they're used a little differently. An array is an ordered list written like this:

Let sonicCharacters = ['sonic', 'tails', 'DrRobotnik']

In an array, each item has a numeric index starting from 0. So, sonicCharacters[0] is "sonic", sonicCharacters[1] is "tails", and sonicCharacters[2] is "DrRobotnik".

An object stores data as a collection of key-value pairs. It looks like this:

Let sonicCharacters = {
main: 'sonic',
bestFriend: 'tails',
enemy:'DrRobotnik'
}

In an object, each piece of data has a key (like main, bestFriend, enemy) and a corresponding value (like sonic, tails, DrRobotnik). The key helps you access the data.

To get data from an array, you refer to the index of the item you want. Since arrays are ordered, the first item has an index of 0. I want to console.log Sonic (because they're my favourite). So I'll write:

console.log(sonicCharacters[0])

To get data from an object, you use the key associated with the value. The key is like a label that helps you find the data you need. To see sonic I would need the write:

console.log(sonicCharacters.name)

Functions

A function in JavaScript is like a little "tool" that does a specific job. You can use it whenever you need it, instead of writing the same code over and over. It's super handy for organising your code and making it easier to read. For example:

function praiseSonic() {
console.log('Sonic is my hero!');
}

Now, whenever you want to say "Sonic is my hero!" you just call praiseSonic() instead of writing the console.log() every time. This isn't a great example because it's not that hard to type console.log('Sonic is my hero!') but you can write some pretty complex functions so you get the point right?
