Blocked on a simple problem

During Dev Academy we were given a set of challenges and one of these challenges was called the flipflop. We were asked to write a function called flipflop that would take a number and simply return the number unless (and here's the hard part) if it was divisible by 3 - it should return flip, if it was divisible by 5 - it should return flop, and if it was divisible by 15 it should return flipflop.

I originally wrote it like this:

And I thought why the heck doesn't this work? It worked for numbers divisible by 3 and 5 but not 15. So I checked what the error was and it said that when there a number divisble by 15 it was returning flip. Hmmmmm? Lets rubber ducky this - if it's returning flip this must mean numbers divisble by 15 are getting the divisible by 3 if statement instead. Ahhhh I see it's becuase it's running through the function in the order I've written it. and obviously if it's divisble by 15 it's also divisble by 3! So all I need to do is put the if statement for numbers divisble by 15 at the start.

And voila! It worked! Sometimes you've just got to think it all through. Think like the code, if you were this code how would you deal with a number divisible by 15?

Being reluctant to ask for help

I've definitely found myself hesitating to ask for help during the Dev Academy course. Mostly because I'm not always sure what exactly I need help with, and there's definitely a bit of embarrassment involved. I want to show someone my code and just ask, “Why isn't this working?” but then I get worried I'll look silly.

But you know what? When it comes to coding, sometimes I really don't know what I'm doing, and that's okay. The only way I'm going to get better is by accepting that I need help sometimes. Plus, I'm sure there are plenty of people in the course feeling the same way.
